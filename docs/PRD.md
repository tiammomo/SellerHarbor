# SellerHarbor 产品需求文档

## 1. 背景

个人和小团队跨境卖家经常同时经营 Temu、TikTok Shop 等多个平台。商品需要在不同平台维护不同 Listing、素材、价格和库存策略，而海外仓往往又是共享的。同一批 SKU 在多个平台销售时，容易出现资料不一致、库存分配不清、低库存预警滞后、热款补货不及时和评价反馈无法沉淀的问题。

SellerHarbor 通过商品主数据、平台映射、轻量仓储库存、评价反馈和 AI 内容能力，把分散的商品运营信息收口到一个跨境卖家商品运营港。当前技术仓库、Docker 服务名、环境变量和 API Header 已统一采用 SellerHarbor 命名。

## 2. 产品目标

- 统一管理商品主档、SKU、变体、平台 Listing 状态和上架素材。
- 管理多平台共享海外仓库存，识别低库存、断货和热款补货风险。
- 追踪好评、评分、销量和热款信号，辅助卖家决定补货、扩平台和优化素材的优先级。
- 保留合规内容生成与审核能力，降低夸大宣传、虚构事实和平台违规风险。
- 为后续平台 API 同步、批量导入导出、库存分配和运营自动化打基础。

## 3. 用户角色

### 跨境卖家 / 店铺运营

需要统一管理 Temu、TikTok Shop 等平台上的商品资料、上架状态、素材和运营动作。

### 仓储与履约负责人

需要看到不同海外仓的 SKU 库存、安全库存、占用库存、低库存预警和平台分配情况。

### 客服人员

需要根据真实服务记录和客户反馈，整理回访话术、评价邀请和售后体验总结。

### 品牌负责人

需要统一语气、审核内容质量、避免不合规表达，并沉淀品牌表达规范。

## 4. 核心功能

### 4.1 商品主数据管理

商家可以录入和维护商品基础信息：

- 商品名称
- 类目
- SKU / SPU
- 变体
- 目标平台：Temu、TikTok Shop 等
- 平台 Listing 状态
- 核心卖点
- 使用场景
- 适用人群
- 材质、规格、口味、颜色等属性
- 物流、售后、服务承诺
- 禁用词和敏感表达

### 4.2 海外仓库存管理

第一阶段先做轻量仓储管理，库存数据可通过商品属性、表格导入或后续 API 同步进入系统。

系统需要支持：

- 仓库名称和国家/区域
- SKU 可用库存
- 平台占用库存
- 安全库存线
- 低库存和断货预警
- 热款库存风险提示
- 多平台共享同一仓库库存的分配视图

### 4.3 真实反馈录入

支持录入客户授权反馈、客服回访摘要、售后记录摘要或平台公开评价摘要。

系统需要区分：

- 已确认事实
- 客户主观感受
- 商家希望强调的卖点
- 不确定信息

系统需要提供确定性的反馈整理能力：

- 从原始反馈中提取摘要、已确认事实、主观感受和不确定表达。
- 结合商品禁用表达和平台风险词输出风险标记。
- 给出推荐用途，例如评价邀请、客服回访、推荐语或详情页口碑。
- 整理结果默认不自动入库，需人工确认后保存为反馈依据。
- 该能力必须不依赖外部模型，保证本地、测试和生产降级场景可用。

### 4.4 内容生成

MVP 默认优先支持低风险、高频运营场景：评价邀请、客服回访、商品推荐语和详情页口碑描述。第一人称体验草稿只在存在真实反馈依据时生成，且默认进入人工审核。

支持以下内容类型：

- 基于真实反馈的评价草稿
- 使用体验文案
- 推荐语
- 评价邀请话术
- 晒单引导话术
- 客服回访话术
- 详情页口碑描述

每次生成可以配置：

- 平台：淘宝、天猫、京东、拼多多、抖音、小红书、独立站等。
- 语气：自然、真诚、活泼、专业、简短、细节丰富。
- 长度：短句、中等、长段。
- 人称：第一人称、第三人称、商家口吻。
- 场景：自用、送礼、复购、新手尝试、家庭使用、办公室使用等。
- 数量：一次生成 1 到 20 个候选版本。

### 4.5 质量评估

生成后自动给出内部评分：

- 事实一致性
- 口语自然度
- 具体度
- 重复度
- 平台风险
- 夸大宣传风险
- 是否像真实用户表达

低分内容需要自动重写或进入人工审核。

### 4.6 审核与导出

运营人员可以：

- 查看生成依据。
- 编辑结果。
- 标记通过、驳回或需要重写。
- 导出纯文本、CSV 或复制到剪贴板。
- 保存为商品常用模板。

## 5. 用户流程

### 单条生成流程

1. 选择商品。
2. 选择内容类型和平台。
3. 输入真实反馈或使用场景。
4. 配置语气、长度和数量。
5. 系统生成候选内容。
6. 系统自动检查风险并打分。
7. 用户编辑并确认。
8. 导出或保存。

### 批量生成流程

1. 上传商品和反馈表格。
2. 选择统一生成策略。
3. 系统逐条生成并检查。
4. 低置信度内容进入待审核列表。
5. 用户筛选、编辑、导出。

## 7. 成功指标

- 商品主档完整度达到 80% 以上。
- Temu / TikTok Shop Listing 就绪率达到 70% 以上。
- 库存健康率达到 85% 以上，热款低库存预警当天可见。
- 运营人员每天能在首页看到明确的下一步动作。
- 内容生成仍保持人工可用率 70% 以上，并持续拦截高风险表达。

## 8. 非目标

- 不做平台自动发布。
- 不做账号矩阵、刷评、刷单或规避平台风控。
- 不做虚假身份、虚假订单、虚假截图生成。
- MVP 阶段不做复杂 CRM 集成。
- 不把公开商品库采集和泛选品 Provider 作为默认主线功能。

## 9. 开放问题

- 首批重点平台是哪几个。
- 首批重点商品类目是什么。
- 是否需要多语言生成。
- 是否需要商家团队权限和审批流。
- 是否需要接入真实订单或客服系统。
