# SellerHarbor 安全与合规边界

## 1. 核心原则

SellerHarbor 的目标是提升商家整理真实口碑内容的效率，而不是制造虚假评价。

所有生成内容都应该满足：

- 有真实商品资料或真实客户反馈作为依据。
- 不冒充真实消费者。
- 不编造订单、使用过程、效果、物流或售后经历。
- 不绕过平台规则。
- 发布前经过人工审核。

## 2. 允许的内容

- 基于客户授权反馈整理的评价草稿。
- 商家口吻的商品推荐语。
- 详情页中的体验描述。
- 客服回访、评价邀请和复购引导话术。
- 对真实反馈进行润色、改写、缩短或多风格表达。
- 对公开评价摘要进行归纳，但需要保留来源和授权边界。

## 3. 不允许的内容

- 冒充买家生成评价。
- 生成虚假购买经历。
- 编造物流、客服、售后或使用效果。
- 大规模生成重复、低质量灌水内容。
- 生成绕过平台风控的策略。
- 生成虚假截图、聊天记录、订单记录或用户身份。
- 编造医疗、保健、金融收益、功效保证等高风险声明。

## 4. 产品内置限制

### 4.1 输入检查

系统需要识别以下意图并拒绝：

- “帮我写 100 条像真实买家的好评”
- “不要看起来像 AI”
- “规避平台审核”
- “编一些不同身份和购买场景”
- “写得像已经使用过”

拒绝后提供合规替代：

- 评价邀请话术。
- 基于真实反馈的待确认草稿。
- 商家口吻推荐语。
- 商品详情页体验描述。

### 4.2 事实约束

每条生成内容必须关联来源：

- 商品资料
- 已确认反馈
- 用户指定场景
- 品牌语气样例

如果无法找到依据，系统应降低表达强度或提示需要补充信息。

### 4.3 风险分级

低风险：

- 普通口碑润色。
- 商品推荐语。
- 评价邀请。

中风险：

- 强效果描述。
- 使用第一人称体验。
- 平台敏感类目。

高风险：

- 伪造用户身份。
- 虚构订单体验。
- 医疗、金融、功效承诺。
- 明确要求绕过平台规则。

### 4.4 审核要求

高风险内容不得自动通过。中风险内容需要人工确认。低风险内容也应保留审核记录。

审核记录包含：

- 审核人
- 审核时间
- 审核结果
- 修改内容
- 风险标记
- 来源依据

## 5. UI 提示

在生成入口附近提示：

```text
请仅输入真实商品信息和真实客户反馈。生成结果为可审核草稿，不应用于伪造消费者评价。
```

在导出前提示：

```text
请确认内容基于真实信息，且符合目标平台规则。
```

## 6. 日志与追溯

系统应保存：

- 用户输入摘要。
- 生成配置。
- Prompt 版本。
- 模型输出。
- 风险检查结果。
- 人工审核记录。

敏感数据需要脱敏存储，尤其是手机号、地址、订单号和聊天记录。

## 7. 平台规则

不同平台对评价、广告、推荐和商家触达有不同要求。产品应预留平台规则库：

- 平台名称
- 内容类型
- 禁用表达
- 重点风险
- 更新日期
- 来源链接

规则需要定期人工更新。系统输出只能辅助判断，不能替代法律或平台合规审核。
