# ReviewPilot 开发路线图

## Phase 0: 项目准备

- 明确首批目标平台和商品类目。
- 确认生成内容的合规边界。
- 确认模型供应商和部署方式。
- 建立基础文档和目录结构。

## Phase 1: LangChain MVP

目标：实现本地可运行的后端生成服务。

范围：

- FastAPI 项目骨架。
- Pydantic schema。
- LangChain 生成链。
- 基础风险拦截。
- 单条内容生成 API。
- 结构化输出。
- 简单内存或文件级存储。
- 单元测试覆盖核心链路。

验收标准：

- 可以录入商品和反馈。
- 可以生成 1 到 5 条候选文案。
- 可以识别明显伪造评价请求。
- 生成结果包含风险标记和来源依据。

## Phase 2: 数据持久化与审核

目标：形成可用的运营闭环。

范围：

- PostgreSQL 数据库。
- 商品、反馈、任务、生成结果表。
- 审核状态流转。
- Prompt 版本记录。
- LangSmith 接入。
- 导出 CSV。

验收标准：

- 用户可以查看历史生成记录。
- 每条文案可以标记通过、驳回或修改。
- 生成链路可以追踪和复盘。

## Phase 3: 前端控制台

目标：让非技术运营人员可用。

范围：

- 商品管理页面。
- 反馈录入页面。
- 生成工作台。
- 审核列表。
- 批量导出。
- 风险标记展示。

验收标准：

- 运营人员无需命令行即可完成单条生成。
- 内容风险和来源依据可视化。
- 结果编辑和导出流程顺畅。

## Phase 4: 批量生成与 RAG

目标：提升规模化运营效率。

范围：

- 批量导入商品和反馈。
- 异步任务队列。
- 品牌语气库。
- 商品知识库。
- 平台规则检索。
- 已通过高质量文案检索。

验收标准：

- 支持批量生成 100 条以上任务。
- 批量任务失败可重试。
- 生成内容能体现品牌语气和商品差异。

## Phase 5: 多租户与团队协作

目标：支持真实商家团队使用。

范围：

- 商家租户隔离。
- 用户、角色和权限。
- 审核流。
- 操作日志。
- API Key 管理。
- 用量统计和配额。

验收标准：

- 不同商家数据隔离。
- 不同角色权限清晰。
- 关键操作可审计。

## 优先级建议

第一周优先做：

- FastAPI 骨架。
- Product / Feedback / Generation schema。
- LangChain 单条生成链。
- 风险拦截 prompt。
- 5 到 10 个测试样例。

先把生成质量和合规链路跑通，再做复杂前端和批量能力。
